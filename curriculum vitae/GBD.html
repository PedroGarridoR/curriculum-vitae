¿EN QUE CONSISTE ESTA ASIGNATURA?

Tratamos el funcionamiento de la base de datos, cual es la organización de una base de datos y que diferentes capas posee.

Para realizar una base de datos se necesita cierta organización de los datos antes de implementarlos en una base de datos al uso, es por ello que antes de empezar a escribir código es necesario hacer esquema conceptual, donde organicemos dichos datos. Este esquema será valido para cualquier modelo que apliquemos a la base de datos.

El esquema que realizamos, se basa en el modelo Entidad/Relación que agrupa los datos en entidades que se relacionan entre sí, y que poseen atributos que los definen.

Después de realizar el esquema conceptual, realizamos el esquema lógico, que depende del tipo de la base de datos, y que no puede usarse para cualquier base de datos, como es el caso del esquema conceputual.

Este esquema lo hemos aprendido a realizar usando el modelo relacional, que básicamente consiste en pasar todos los datos escritos en el modelo E/R, a tablas.

Cada entidad forma una tabla, que a su vez se relaciona con otra tabla, dependiendo del tipo de cardinalidad que tenga dicha relación.

Por último, y una vez realizado ambos esquemas, pasamos al Interno, en el cual escribimos el código en lenguaje MSQL empleando el programa Datagrip.
